## 1. 全体を一言で

**「市場シミュレーションモデルの主張を、ちゃんと審査する方法を作り、それが機能することを証明し、実際に審査して見せる」**——研究プログラム全体は「審判業の立ち上げ」。

背景: エージェント型の市場モデル(プログラムされた売買エージェントを多数走らせて市場を再現するシミュレーション)の世界では、「俺のモデルは本物の市場データっぽい統計的なクセを再現する。だから市場の仕組みを説明できている」という論法が 30 年使われてきた。**この論法はザル**——全然違う仕組みのモデルでも同じ「クセ」は出せるから。ザルだと何が困るかというと、LLM エージェントのシミュレーションが大量生産される時代に、誰の主張も審査できない。

そこで: (1) ザルであることを言い逃れできない形で証明する、(2) 代わりに効く審査方法を出す、(3) その審査方法で実際に有名研究を審査して見せる。

なぜ分けるか: 道具作りと中身の研究を混ぜると両方歪む(道具が一つの研究テーマに引きずられる/研究が道具磨きに吸われる)。だから repo ごと分離した。

時系列の注意: **隣(P2)が先に結果を出している**。審査の「精神」(合否基準を結果より先に公開固定する、負けたときの言い分も先に書く)は道具が完成してなくても運用できるから、P2 で先に実運用した。このリポの道具(P1)はまだ証明実験の途中。

## 3. 三本の論文を平文で

**P1(このリポ)**: 問い =「モデルが本物っぽい統計的クセを再現できることは、モデルの中身が正しいことの証拠になるか?」。示しに行く答え =「ならない。クセが完全に同じになるよう調整した、中身の違うモデルたちを実際に作れる(=見た目では区別不能)。でも、エージェントが見ている情報をわざと曇らせて市場の反応を見ると、中身の違いが浮き出る(=介入すれば区別できる)」。つまり**見た目の審査は落第、介入の審査は合格**、を一つの実験セットで示す。これが審判の笛が機能することの証明。

**P2(隣のリポ)**: 問い =「『AI 価格設定は放っておくと勝手に談合する』という有名な研究(Calvano 2020)は、本物の市場に近い環境でも成立するか?」。本物の市場では儲けの機会がまばらで運任せ(毎回必ず儲かる教科書環境と違う)。出た答え =「ほぼ成立しない。『これは談合だ』と機械判定できたのは 18 条件中 1 つだけで、それは現実から遠い特殊な隅(イベントが異常に多く、学習が異常に速く、しかも談合を壊す捕食者を切った世界)だった」。判定基準は結果を見る前に公開登録済みなので、後出しと言われない。

**P3(将来、まだ無い)**: 問い =「有名 LLM エージェント市場シミュ論文の主張は、審査に耐えるか?」。手順 = 再現できるか → 結果が偶然(乱数のクセ)でないか → 効果が主張どおりの仕組みから来ているか。**P1 と P2 の実績ができるまで着手しない**(実績なしでやると「無名研究者の言いがかり」として潰されるから。順序は固定)。

## 4. いま終わっていること / 次やること

終わっていること:
- P2: 実験の本体が完了。主結果(上記)が出て、公開事前登録(osf.io/63pj2)も通した。残りは頑健性チェックと論文執筆。
- P1: 戦略文書と論文の設計図は固まった。モデルと測定器の実装は半分(クセの測定器は動く、「クセを同じにする調整」を探索中)。

次やること(順番どおり):
1. **Yuito**: 開いている PR を読んで merge 判断(読む順と平文要約は PR の description 冒頭と、直近のチャットにある)。
2. **Yuito**: 別の会話で書いた「本実験の事前登録文書(prereg_paper_grade_v0)」をこのリポに commit する(現状、正典のはずの文書がどこにも保存されていない)。
3. toy 実験(小規模パイロット)を最後まで走らせ、合否(GO/PARTIAL/PIVOT)を出す。**PIVOT(全敗)なら P1 論文は書かない**——その場合どう撤退するかも先に文書化済み。
4. 合格なら: 本実験の事前登録 → 本実験 → P1 論文。並行して P2 論文を書く。

## 6. 用語集(専門語 → 平文)

| 専門語 | 平文 |
|---|---|
| ABM | エージェント型モデル。プログラムされた売買主体を多数走らせるシミュレーション |
| stylized facts(SF) | 実際の株価データに共通する統計的なクセ(値動きの荒い時期が固まる、極端な値動きが教科書の想定より多い、等) |
| SF battery | そのクセを測る測定器一式(クセ 4 種 = SF1-4 を数値化する) |
| SF-等価 / 等価集合 | 「クセの測定値では区別がつかない」状態にツマミ調整したモデルの集まり |
| calibration | そのツマミ調整 |
| 機構(mechanism) | モデルの中身の仕組み(例: 値動きを追いかける型 vs 周りの真似をする型) |
| T / H | このリポで作った 2 つの模型。T = 値動き追随型、H = 群れ型。どちらも教科書モデルの部品の単離 |
| CB / LM / ALW | 分野で 30 年読まれてきた教科書モデル 3 つ(Cont-Bouchaud / Lux-Marchesi / Alfarano-Lux-Wagner) |
| ZI | ゼロ知能。ランダム注文を出すだけの対照群 |
| Genoa(-ZI+) | 戦略ゼロなのにクセを再現してしまう既知のモデル。入れば「見た目審査のザルさ」の最強の証拠 |
| 介入応答(IR) | エージェントの見ている情報をわざと曇らせ、市場全体の挙動がどう変わるかを測ったもの |
| 観測チャネル / B2 | 「エージェントが何を見ているか」の通り道。介入はここだけに行う(モデルの中身そのものは触らない) |
| masking / scheme / θ | 曇らせ方(平均でぼかす、遅らせる、ノイズを混ぜる等の 4 方式)とその強さ(θ = 0〜1) |
| classifier / accuracy | 2 つのモデルの出力を機械学習に見分けさせたときの正答率。50% = 区別不能、100% = 完全に区別 |
| TOST | 「区別できない」を統計の作法で言い切るための検定(言いっぱなし禁止の道具) |
| 検出力 / MDE / Holm | 「もし差があるなら見逃さない」を保証する実験サイズの設計と、検定を何十回もやる時の偽陽性対策 |
| 事前登録(prereg / OSF) | 合否基準・解釈・負けたときの言い分を、結果を見る前に公開タイムスタンプつきで固定すること |
| 縮退規則 | 結果が負けだったとき、主張をどこまで引っ込めるかの事前の取り決め |
| toy / GO / PARTIAL / PIVOT | 小規模パイロット実験とその合否(合格/部分合格/全敗=撤退) |
| paper-grade | 論文に載せる本実験(パイロットとは別に、改めて事前登録してから走らせる) |
| Null-0/1/2 | 計器のゼロ点確認 3 種(0: 構造的にゼロのはずの被験体でゼロが出るか、1: 同じモデル同士が 50% になるか、2: 中身が同じでツマミだけ違う場合に反応しないか) |
| Model Contract | 模型を審査台に載せるための差込口の規格(6 つの操作を実装すれば誰のモデルでも審査できる) |
| provenance / L2 | 全実験 run の身元記録(誰が・どの設定で・どの乱数で走らせたか。再現の保証) |
| claim-first | 「言い切りたい一文」を先に決め、それに必要十分な実験だけやる進め方 |
| Calvano | 「AI 価格設定が勝手に談合を学習する」を示した有名研究(2020) |
| 認定(certify) | P2 で「これは談合と呼んでよい」の機械判定(基準 3 つを全部満たす場合のみ) |
| committed / revisable | P2 の市場設計の 2 方式(談合を壊す捕食者が居る世界/居ない世界、に対応) |
| BCS 較正点 | 実在市場(シカゴの先物)の実測値に合わせたパラメータ。「現実はどこか」を地図に置くピン |
